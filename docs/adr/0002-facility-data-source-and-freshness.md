# ADR-0002 施設データの出典と鮮度管理

- Status: Proposed
- Date: 2026-07-12

## Context

AI推薦サービスは、施設の住所だけでなく、営業時間、料金、利用可能競技、初心者適性、照明、屋根、防具ルールなどを必要とする。

スケート施設の利用可否や営業時間は変化するため、未確認の情報を推薦すると、利用者が現地で滑れない、ルール違反になる、サービスを信頼しなくなる可能性がある。

## Decision

以下のデータ方針を採用候補とする。

1. 初期は開発者が確認できる1都市圏20〜30施設から開始する。
2. 自治体・施設の公式ページ、公式案内、利用規約を優先する。
3. 施設ごとに`source_url`、`source_type`、`verified_at`、`status`を保存する。
4. 利用可否不明のストリートスポットはMVPに含めない。
5. Instagramや検索結果からの無断スクレイピングを前提にしない。
6. 全国557施設への拡大は、初期検証とデータ更新手順が成立した後に行う。
7. AIはデータを新規生成せず、検証済みデータの検索・説明・翻訳に限定する。

## Alternatives

### 557施設を最初に一括登録する

全国価値を早く表現できるが、情報確認と更新の負担が大きく、検証前にデータ作成で停止する可能性が高い。

### ユーザー投稿だけでデータを作る

投稿コストを分散できるが、コールドスタート、誤情報、違法スポット、秘密スポットの公開、モデレーションが問題になる。

### Google Maps等の外部情報だけを利用する

住所・経路には使えるが、スケートボードのレベル、路面、セクション、ルール、初心者適性が不足する。

## Consequences

### Positive

- 推薦理由の根拠を説明できる
- 施設情報の更新責任を明確にできる
- AIの誤回答を抑制できる
- 利用者が情報の鮮度を判断できる

### Negative

- 初期データの手動作成が必要になる
- 全国展開まで時間がかかる
- 施設ごとに情報の粒度が揃わない可能性がある

## Open Questions

- 情報の再確認周期を施設属性ごとにどう設定するか
- 利用者の訂正報告を誰が承認するか
- 民間施設の掲載許諾をどのように取得するか
- 施設情報の英訳を誰が確認するか
