# ADR-0003 AI導入前にルールベース推薦を作る

- Status: Accepted
- Date: 2026-07-12

## Context

spot-diggzの価値は、利用者の目的・気分・レベル・時間・現在地・交通手段から、適切な施設を提案することである。

営業時間、利用可否、距離、料金、防具ルールなどは、正確性と説明可能性が必要である。AIに判断を任せると、データにない情報を補完して断定する可能性がある。

## Decision

推薦処理を次の2層に分ける。

### Deterministic Layer

- 営業時間内か
- 利用可能な競技か
- 目的地までの移動条件を満たすか
- 初心者等の必須条件を満たすか
- 料金や屋内外等のハード条件を満たすか

### AI Layer

- 自然文を入力条件へ変換する
- 検索結果を利用者向けに説明する
- 候補を比較する
- 検証済み施設情報を翻訳・要約する

最終的な施設選定はDeterministic Layerの結果を正とする。

## Alternatives

### AIに施設選定をすべて任せる

自然な会話は実現しやすいが、根拠のない推薦や古い情報の断定を防ぎにくい。

### 固定条件だけで検索する

初期実装は簡単だが、利用者の目的や気分に合わせた説明・対話ができない。

## Consequences

### Positive

- 推薦理由を説明できる
- テストしやすい
- AIプロバイダーを変更しやすい
- AIが停止しても基本検索を提供できる
- 誤情報の原因を施設データ、条件判定、AI説明に分けて調査できる

### Negative

- 施設データの構造化が必要になる
- 自然文入力と推薦処理の連携が必要になる
- AI機能の見た目上の魅力を後から追加することになる

## Follow-up

- まず選択式フォームとルールベース推薦を実装する
- 推薦結果に必ず根拠と確認日を持たせる
- AI導入時は、未確認情報を断定しない評価ケースを作成する
